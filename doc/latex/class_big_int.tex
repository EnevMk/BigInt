\doxysubsection{Detailed Description}
The main class for the whole project. 

Class \mbox{\hyperlink{class_big_int}{Big\+Int}}, its purpose being enabling users to work with numbers too big that aren\textquotesingle{}t supported by the language built-\/in types. Working in a billion number system, it offers broad functionality along with great effectiveness resource-\/wise.

Core functionalities are\+: Addition, Subtraction, Multiplication and a fast exponentiation alghoritm.

Paramater Constructor, initializing a \mbox{\hyperlink{class_big_int}{Big\+Int}} number by \mbox{\hyperlink{class_string}{String}}. 

This is the easiest and most intuitive way of initializing an object of Class \mbox{\hyperlink{class_big_int}{Big\+Int}}. Given that the library uses a not-\/so-\/popular billion number system, the developers want to use it should conform to some standarts. Since a single digit in a billion number system is in the range \mbox{[}0 -\/ 999999999\mbox{]}, \mbox{\hyperlink{class_big_int}{Big\+Int}} numbers can look messy at first. So, the developers have decided that \mbox{\hyperlink{class_big_int}{Big\+Int}} numbers should be represented with a space between every {\itshape billion system digit} and no spaces between single (decimal) digits. Therefore, strings passed to that constructor should stick to that standart. \hypertarget{class_big_int_autotoc_md0}{}\doxyparagraph{Example}\label{class_big_int_autotoc_md0}
\mbox{\hyperlink{class_big_int}{Big\+Int}}(\char`\"{}134 240000\char`\"{}); // This Creates the \mbox{\hyperlink{class_big_int}{Big\+Int}} number 134 240000, two digits, which in decimal is 134 000 240 000.~\newline
\mbox{\hyperlink{class_big_int}{Big\+Int}}(\char`\"{}5 1\char`\"{}); // This creates the \mbox{\hyperlink{class_big_int}{Big\+Int}} number 5 1, that\textquotesingle{}s two digits, which in decimal is 5 000 000 001~\newline
~\newline
 The constructor supports a bit of validation, like removing white\+Spaces. \hypertarget{class_big_int_autotoc_md1}{}\doxyparagraph{Example}\label{class_big_int_autotoc_md1}
\mbox{\hyperlink{class_big_int}{Big\+Int}} big\+Number(\char`\"{}13     0   99391\char`\"{}); // Creates number 13 0 99391, three digits, 13 000 000 000 000 099 391 -\/ thirteen quintillion something..~\newline
\mbox{\hyperlink{class_big_int}{Big\+Int}}(\char`\"{}900000000 0\char`\"{}) // 900 000 000 000 000 000 -\/ nine hundred quadrillion.~\newline
~\newline
Passing an invalid string though (cont. symbols other than numbers) causes the program to stop.~\newline
Therefore developers who use this library in their project should think of implementing proper validation.
